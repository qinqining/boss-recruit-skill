# 简历筛选评分任务

与 Obsidian 测试时相同的目录分工；**不依赖 Obsidian**，Claude Code 直接读本地 Markdown/PDF 即可。

| 目录/文件 | 用途 |
|-----------|------|
| `技术SEO岗位要求.md` | 岗位 JD 与硬性/加分项，打分的依据 |
| `简历筛选任务.md` | 给 Agent 的流程说明（读谁、输出格式、如何归档） |
| `任务笔记.md` | HR 随手记（可选） |
| `需筛选简历/` | 待打分候选人简历（建议每人一个 `.md`） |
| `输出/` | 评分结果 Markdown，**文件名含日期**：`简历筛选评分结果_YYYY-MM-DD.md`；同日多份加 `_HHmm` |
| `优秀简历案例/` | 正例 PDF（脱敏后再考虑是否提交 Git） |
| `不合格简历案例/` | 反例 PDF（同上） |
| `已归档/` | 已处理简历：**按日期建子目录** `已归档/YYYY-MM-DD/` 再存放候选人文件 |

隐私：`需筛选简历/`、`输出/`、`已归档/` 已在仓库 `.gitignore` 中默认忽略内容，仅保留目录占位。案例 PDF 若含真实信息，请勿提交远程仓库。
