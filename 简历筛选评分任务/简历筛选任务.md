# 简历筛选任务 — 给 Claude Code 的执行说明

## 输入

1. 阅读 `SEO运营岗位要求.md` 作为唯一岗位依据（若有多岗位，再增加对应 JD 文件并在此列出）。
2. 浏览 `优秀简历案例/`、`不合格简历案例/` 中的 PDF（若有），对齐「好/差」风格，**不得**从案例中抄真实姓名与联系方式到新输出。
3. 对 `需筛选简历/` 下每个候选人材料逐一评分。材料为 **PDF** 时，先 `py scripts/pdf_resume_to_md.py <文件>.pdf` 得到同名 `.md` 再读（`pip install pymupdf`）；扫描件需 OCR。优先以 **Markdown 文本** 为 Agent 输入。

## 输出

- 在 `输出/` 下写入本轮结果文件，**文件名必须以日期为主**（便于排序与检索）：
  - 默认：`简历筛选评分结果_YYYY-MM-DD.md`（`YYYY-MM-DD` 为报告覆盖的筛选发生日或定稿日，与团队约定一致即可）。
  - **同一天多份报告**：不得覆盖已有文件；在后缀加时间 `简历筛选评分结果_YYYY-MM-DD_HHmm.md`（24 小时制，如 `2026-05-13_1530`）。
- 报告文首建议写一行**报告日期**（与文件名日期一致），便于 HR 核对。
- 每条候选人至少包含：**姓名或文件名**、**总分**、**各维度分**、**结论**（通过 / 待定 / 不通过）、**简短理由**（可引用简历要点，勿编造经历）。

## 归档（可选）

- 评分完成后，将已从 `需筛选简历/` 处理完毕的简历原件移入 **`已归档/YYYY-MM-DD/`**（按**归档操作当天**或**筛选批次日期**二选一，团队内固定一种即可；推荐与报告文件名使用同一逻辑）。
- 若目录不存在则创建，例如 `已归档/2026-05-13/郭嘉慧.md`；**移动前**确认 HR 无需再改源文件。

## 注意

- 候选人材料含隐私，勿粘贴到外部未授权渠道。
- 与 BOSS 自动化脚本（`scripts/greet.py` 等）无强绑定；本任务以本地 Markdown/PDF 为准。
