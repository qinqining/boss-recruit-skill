# 岗位层级说明（素质表 vs 当前招聘）

## 背景

- 素质表以 **`面试方案生成任务/岗位素质项/`** 下 `01_`、`02_` Markdown 为 Agent 输入。
- **当前实际招聘岗位：SEO 助理** — 更侧重 **执行配合、学习速度、基础工具与数据意识、沟通闭环**，一般不应按「成熟运营负责人」标准一刀切。

## 给 HR 的一句话

素质表**继续用作「考察维度与题库母题」**，但面试官心里要有 **助理档录用线**：多数题落在「能否在指导下把事做对、学明白、跟得上」，而不是「能否独立定战略、带大项目」。

## 给 Claude Code 的硬性约定

生成 `面试方案生成任务/输出/` 下方案时，若用户未提供另一份「助理专用 JD」，则默认 **当前面试岗位 = SEO 助理**，并遵守：

1. **文首声明**（放在方案日期下一行）：  
   `实际岗位：SEO助理；素质表文件为 SEO运营 档，本题集已按助理向降维校准。`
2. **分档怎么用**：表中 D–G 列以 **4–6 分（胜任）** 为主要对标区间，描述「助理在带教下应达到的证据」；**7–10 分** 仅作加分项追问，**不得**把 9–10 分档写成「默认录用门槛」。
3. **题干怎么改**：`02_素质分档与面试母题.md` 中高优先级母题 **意图保留**，但表述改为助理语境，例如：  
   - 少问「你如何定全年战略 / 如何统筹多线」；  
   - 多问「上级给目标后你如何拆解执行」「你如何记录与反馈数据」「你如何请教师傅或跨部门同事」「你学过哪些工具、犯过什么错如何改」。
4. **JD 与 `01_岗位与JD门槛.md`**：若其中职责明显偏「全责运营」，方案中须有一句说明：**表格 JD 为运营全责描述，本题集已收缩为助理职责范围**；具体题项以「协助、参与、在监督下完成」为主。
5. **若日后提供** `SEO助理岗位要求.md`（或用户指定路径），则以该文件为 **实际录用门槛**，素质表退居「维度与母题来源」。

将此文件与 `任务.md` 一并作为面试方案生成的前置阅读。
